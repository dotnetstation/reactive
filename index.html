Nothing new, but usefull concepts

What it is Reactive Programming

  • Reactive programming is programming with asynchronous data streams.
  • With the functional spirit with functions to combine, create and filter any of those streams

It's also about side effects

Sample with Rxjs:


var requestStream = Rx.Observable.just('https://api.github.com/users');

var responseStream = requestStream
  .flatMap(function(requestUrl) {
    return Rx.Observable.fromPromise(jQuery.getJSON(requestUrl));
  });

responseStream.subscribe(function(response) {
  // render `response` to the DOM however you wish
});

Functions Rxjs:


/* Get stock data somehow */
var source = getStockData();

source
  .filter(function (quote) {
      return quote.price > 30;
  })
  .map(function (quote) {
      return quote.price;
  })
  .forEach(function (price) {
    console.log('Prices higher than $30: $' + price);
  });

Reactive UI with Reactjs

Reactjs philosophie

  • Simple made easy
  • Rethinking best practices
  • Functional thoughts (immutability, declaratif, composition...)

Rethinking best practices

  • Throwing out the notion of templates and building views with JavaScript
  • “Re-rendering” your entire application when your data changes
  • lightweight implementation of the DOM and events
(Data) => View

Fast, really fast

  • Virtual Dom
  • Rerendering the entire view, but apply only changes

Isomorphic

  • Generate dom in the server side
  • Regenerate it everytime you need it, in the client side
  • You win fast rendering, better seo and client refresh
